Introduction to BST

A Binary Search Tree (BST) is a specialized type of binary tree that maintains its elements in a sorted order. Each node contains a unique value, and the tree satisfies the following properties:

  • All nodes in the left subtree of a node have values less than the node.
  • All nodes in the right subtree of a node have values greater than the node.
  • Both left and right subtrees are also BSTs.

This structure ensures that searching, insertion, and deletion operations can be performed efficiently, generally in O(log n) time for balanced trees.

BSTs are widely used for fast data retrieval and form the foundation for advanced data structures like AVL Trees and Red-Black Trees. Common use cases include:

  • Implementing dynamic sets and dictionaries.
  • Maintaining sorted sequences of data efficiently.
  • Supporting database indexing and search operations.
  • Performing range queries and nearest-neighbor searches.

Tree traversals are methods to access elements in different orders:

  • Inorder: Left → Root → Right (gives ascending order).
  • Preorder: Root → Left → Right (useful for copying the tree).
  • Postorder: Left → Right → Root (useful for deleting nodes).

This project visualizes a BST from user input, allowing users to see how nodes are added and how different traversal methods work. By visualizing these operations, learners can easily understand the behavior and structure of BSTs.
