A detailed look into Binary Search Trees and their tree traversals: Inorder, Preorder, and Postorder.

What is a Binary Search Tree?

Definition: A Binary Search Tree (BST) is a node-based data structure where:

  • The left subtree of a node contains only nodes with values less than the node.
  • The right subtree contains only nodes with values greater than the node.
  • Both left and right subtrees must also be BSTs.

Applications: Efficient searching, insertion, deletion, and forming the basis for advanced trees (AVL, Red-Black Tree).

Tree Traversals

// Inorder Traversal (Left → Root → Right)
function inorder(node) {
    if(node != null) {
        inorder(node.left);
        console.log(node.data);
        inorder(node.right);
    }
}

// Preorder Traversal (Root → Left → Right)
function preorder(node) {
    if(node != null) {
        console.log(node.data);
        preorder(node.left);
        preorder(node.right);
    }
}

// Postorder Traversal (Left → Right → Root)
function postorder(node) {
    if(node != null) {
        postorder(node.left);
        postorder(node.right);
        console.log(node.data);
    }
}

Step-by-Step Example

Consider the BST built from inserting: 50, 30, 70, 20, 40, 60, 80

Inorder: 20 → 30 → 40 → 50 → 60 → 70 → 80 (Sorted order)

Preorder: 50 → 30 → 20 → 40 → 70 → 60 → 80

Postorder: 20 → 40 → 30 → 60 → 80 → 70 → 50
